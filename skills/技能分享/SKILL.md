# 技能分享 — 脱敏打包 Claude Code 技能

将任意 Claude Code 技能脱敏、打包为独立文件夹，附带配置文档，可直接发给朋友使用。

> ⚙️ **使用前先列一份「你自己的敏感标识清单」**：本机用户名、姓名（中/英/拼音）、机构/学校、导师/同事名、邮箱域名、内网网段、NAS/HPC 专有路径、自有域名/公网 IP、你给某些 API 供应商起的内部代号。下面流程里凡是出现 `<...>` 占位的地方，都用你这份清单里的真实值去扫。

## 触发方式

- "分享 XXX 技能" / "把 XXX 打包给朋友" / "脱敏 XXX"
- 直接给技能路径："/技能分享 image-generate"

## 输入

- **技能名称或路径**（必填）：技能 .md 文件名、commands 下的相对路径、或绝对路径
- **输出位置**（可选）：默认 `~/Desktop/{技能名}-kit/`
- **目标 API 供应商**（可选）：如 "只保留某指定供应商"，其余供应商分支删除

## 执行流程

### Step 1: 定位技能及关联文件

1. 读取技能 `.md` 主文件
2. **递归追踪所有引用**：
   - 文档中 `路径`、代码块中的文件引用
   - 同目录下的 `.py` / `.sh` / `.yaml` / `.csv` / `.json` 等
   - `scripts/`、`templates/`、`references/` 子目录
   - symlink 指向的实际文件
3. 列出完整文件清单，确认哪些需要打包

### Step 2: 敏感信息扫描

对所有文件执行敏感信息扫描。**必须检测的类别**：

#### 2.1 凭证类（最高优先级）

| 类型 | 识别模式 | 处理方式 |
|------|----------|----------|
| API Key | `sk-`、`key-`、`token=`、长随机字符串在注释/字符串中 | 删除值，保留变量名 |
| 密码 | `密码:`、`password:`、`passwd`、`sshpass -p` | 替换为 `<YOUR_PASSWORD>` |
| Token/Secret | `token`、`secret`、`bearer`、base64 长串 | 替换为占位符 |
| SSH 密钥路径 | `~/.ssh/id_rsa`、`IdentityFile` | 替换为通用路径 |

#### 2.2 个人信息

| 类型 | 识别模式 | 处理方式 |
|------|----------|----------|
| 用户名 | 你的本机账户名 / 拼音名 / 中文名 | 替换为 `<YOUR_USERNAME>` |
| 邮箱 | 你的机构 / 个人邮箱域名（如 `@xxx.edu`、`@gmail.com`） | 替换为 `your@email.com` |
| 团队成员 | 同事 / 组内人名、成员列表 | 整段删除或替换为示例名 |
| 组织信息 | 你的机构 / 学校 / 导师 / 团队名 | 删除或泛化 |

> ⚠️ **大小写陷阱**：人名占位别只扫小写。比如英文名既可能是 `john` 也可能是 `John`，grep 记得 `-i` 或两种都列。光靠 grep 容易漏，**务必通读全文**一遍。

#### 2.3 基础设施

| 类型 | 识别模式 | 处理方式 |
|------|----------|----------|
| 内网 IP | `172.16.x.x`、`192.168.x.x`、`10.x.x.x`（RFC1918 私有网段） | 替换为 `<YOUR_SERVER_IP>` |
| 公网 IP/域名 | 特定服务器 IP、自有域名 | 替换为 `<YOUR_SERVER>` |
| SSH 配置 | `Host xxx`、`User xxx`、端口号 | 泛化为模板 |
| NAS/HPC 路径 | NAS 数据卷专有前缀（如 QNAP 默认 `/share/CACHEDEV1_DATA/`）、HPC 专有路径 | 替换为 `<YOUR_NAS_PATH>` 等 |
| 家目录路径 | `/Users/<你>/...`、`~/<你的私人目录>/...` 等含个人目录结构的绝对路径 | 替换为 `~/...` 或占位 |

#### 2.4 订阅/服务

| 类型 | 识别模式 | 处理方式 |
|------|----------|----------|
| 订阅链接 | VPN clash 订阅 URL、含 token 的链接 | 整行删除 |
| 付费服务账号 | 特定平台的用户名/ID | 删除或替换 |

### Step 3: 脱敏并生成文件

对每个文件：

1. **复制原始内容**
2. **执行替换**（按 Step 2 的规则）
3. **移除不需要的供应商/功能**（如用户指定只保留某供应商）
4. **简化路径**：私有安装路径（如 `~/.claude/image-api/`）→ 使用打包目录内的相对引用，README 中建议通用安装路径
5. **清理自引用**：去掉指向你私有系统的 symlink 说明、旧路径兼容等

### Step 4: 脱敏验证（关键步骤）

**在写入文件后，必须用 Grep 对输出目录做全量扫描**（把下面占位换成你自己的真实值）：

```
扫描正则（必须全部为零命中）：
- 真实 API Key 片段（取前 10 字符）
- <你的用户名>|<你的拼音名>|<你的中文名>（大小写都扫，grep -i）
- <你的机构英文名>|<你的机构中文名>
- <你的导师名>|<同事/组内人名>
- 172\.16\.|192\.168\.|<你的具体内网 IP>
- <你的具体公网 IP/自有域名>
- <你的私人目录名>|/Users/<你>（家目录路径）
- <已移除的 API 供应商代号>（如适用）
- 任何在 Step 2 中识别到的原始敏感值
```

如果有命中 → 修复后重新扫描，直到干净。

### Step 5: 生成配置文档

创建 `README-配置指南.md`，结构如下：

````markdown
# {技能名} - 配置指南

{一句话说明这个技能做什么，价格/免费}

---

## 1. 前置条件
{需要注册的服务、获取的 API Key、安装的软件}

## 2. 配置环境变量
{需要设置的环境变量，给出 zshrc/bashrc 模板}

## 3. 安装依赖
{pip install / brew install / npm install 等}

## 4. 放置文件
{建议的安装路径，cp 命令}

## 5. 测试
{一条可直接跑的测试命令}

## 6. 配置 Claude Code 技能
{把 .md 放到 commands/，在 CLAUDE.md 中加引用}

---

## 使用示例
{3-5 个典型用例}

## 参数速查
{表格}

## 常见问题
{3-5 个 Q&A}
````

### Step 6: 输出确认

向用户报告：
1. 输出文件夹路径和文件列表
2. 脱敏扫描结果（确认零命中）
3. 朋友拿到后的操作步骤摘要

## 注意事项

- **宁可多脱不可少脱**：拿不准是否敏感，一律替换为占位符
- **保留功能完整性**：脱敏后的代码必须能正常运行（朋友填入自己的 Key 后）
- **不打包无关文件**：PPT 生成器、combine 工具等如果与核心技能无关，不打包
- **中文路径安全**：输出文件夹名用英文或拼音，避免跨平台问题
- **README 用中文写**：朋友是中文用户

## 文件结构模板

```
{技能名}-kit/
├── README-配置指南.md        # 朋友看的配置文档
├── {技能名}.md               # Claude Code 技能文档（脱敏版）
├── {主脚本}.py / .sh         # 核心代码（脱敏版）
├── {配置文件}.py / .yaml     # 配置文件（脱敏版，占位符）
└── {其他必要文件}            # 模板、数据文件等
```
