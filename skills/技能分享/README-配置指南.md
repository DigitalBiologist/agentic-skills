# 技能分享 — 配置指南

一个**元技能**:让你的 agent 把任意 Claude Code 技能**脱敏打包**成独立文件夹,附带配置文档,可以直接发给朋友、或推到公开仓库。它会递归追踪技能的关联文件、扫描敏感信息(API Key / 密码 / Token / 用户名 / 邮箱 / 团队组织 / 内网 IP / 订阅链接)、替换为占位符,最后用 grep 全量自检确认零命中。

**纯指令技能,零依赖、零成本**,不调用任何 API。

---

## 1. 前置条件

无。只要你的 agent 能读写本地文件、能跑 `grep` 即可(Claude Code 自带)。

## 2. 配置环境变量

无。

## 3. 安装依赖

无需安装。

## 4. 放置文件

```bash
# Claude Code(文件夹形式)
cp -r 技能分享 ~/.claude/commands/

# 或只取单文件当 slash command
cp 技能分享/SKILL.md ~/.claude/commands/技能分享.md
```

## 5. 测试

新开 session 后:「把 XXX 技能脱敏打包给朋友」或「/技能分享 XXX」。agent 会:定位技能及其关联文件 → 扫描并替换敏感信息 → grep 自检零命中 → 生成 `README-配置指南.md` → 报告输出路径。

## 6. 配置为技能

放进 `~/.claude/commands/` 后自动发现(凭 `SKILL.md` 的 frontmatter)。触发词:「分享 XXX 技能 / 把 XXX 打包给朋友 / 脱敏 XXX」。

---

## ⭐ 关键:先建「你自己的敏感标识清单」

这个技能脱敏的质量,取决于你有没有先告诉它**你的**敏感标识长什么样。第一次用前,在心里(或写给 agent)列清楚:

| 类别 | 你的值(示例) |
|------|--------------|
| 本机用户名 | `你的 macOS/Linux 账户名` |
| 姓名 | 中文名 / 拼音 / 英文名(大小写都算) |
| 机构 / 学校 | 中英文 |
| 导师 / 同事名 | 列出来 |
| 邮箱域名 | `@你的机构.edu`、个人邮箱 |
| 内网网段 | `172.16.x.x` / `192.168.x.x` / `10.x.x.x` |
| NAS / HPC 路径 | 你的专有路径前缀 |
| 公网 IP / 自有域名 | 列出来 |
| API 供应商内部代号 | 你给某些服务起的私人昵称 |

agent 据此扫描,而不是只靠内置的通用模式。

## 使用示例

1. **打包单个技能发朋友** —— "把 image-generate 技能脱敏打包",产物在 `~/Desktop/image-generate-kit/`,直接压缩发出。
2. **只保留某一个供应商** —— "脱敏 X 技能,只保留 OpenAI 那条路径",其余供应商分支删掉。
3. **推公开仓库前的过滤** —— 推任何技能到 GitHub 前先跑一遍,确保不带个人信息进 git 历史。

## 常见问题

**Q: grep 自检零命中了,就一定干净吗?**
A: 不一定。grep 只抓你列进清单的模式,**大小写、变体、家目录路径(`~/<你的私人目录>`)很容易漏**。SKILL.md 的 Step 2 反复强调:grep 之外**务必通读全文**一遍。

**Q: 脱敏后技能还能用吗?**
A: 能。敏感值被换成 `<YOUR_XXX>` 占位,朋友填入自己的值即可运行。逻辑、脚本、模板都保留。

**Q: 哪些文件不该打包?**
A: 与核心功能无关的(PPT 生成器、临时 combine 工具等)、含凭证的 `.env`、私有 symlink 指向的系统文件。

**Q: 中文技能名做文件夹名安全吗?**
A: git/GitHub 支持 UTF-8,macOS/Linux 没问题;若要发 Windows 用户或图稳妥,输出文件夹名可用英文/拼音。
