# 虚拟细胞文献网络

把多篇论文连接成面向虚拟细胞建模的证据网络。最终产物必须是交互式 markmap HTML。

## 输入

- 两篇或多篇论文 PDF、速读页、脑图、全文文本或用户给出的论文摘要。
- 可选：中心主题，如嘌呤代谢、鸟苷生产、中心碳流、抗生素敏感性、蛋白互作网络。
- 可选：目标模型或任务，如 `E. coli iML1515`、FBA 假设、调控网络、酶复合体机制。

## 输出

```text
<输出目录>/
└── literature_network_markmap.html
```

HTML 必须包含：

- 交互式 markmap。
- 工具栏：全部展开、全部收缩、显示/隐藏详解、L1-L5。
- 多论文总览：每篇论文的角色。
- 共同实体表述：基因/蛋白、代谢物、通路、表型。
- 综合建模假设：哪些证据能合并，哪些不能硬连。

## 工作流

1. **建立文献角色**
   - 给每篇论文分配角色：工程应用、综述背景、机制验证、模型方法、组学数据、参数来源。
   - 写明每篇论文对虚拟细胞模型的贡献。

2. **抽取共同实体**
   - genes/proteins：同名、同义名、跨数据库名称。
   - metabolites：底物、前体、中间体、产物、辅因子。
   - pathways：通路和子通路。
   - phenotypes：产量、生长、适应度、毒力、抗性、能荷。
   - methods：FBA、BACTH、transcriptomics、metabolomics、fermentation 等。

3. **建立连接类型**
   - support：一篇论文支持另一篇论文的机制或假设。
   - extend：一篇论文把另一篇的结论扩展到新层次。
   - constrain：一篇论文给模型设置边界或限制。
   - conflict：结论不一致或物种/条件不同，不能直接合并。
   - gap：连接中缺少参数、反应 ID、实验验证或动态机制。

4. **沉淀综合建模假设**
   - 每条假设要列出支持文献、涉及实体、模型表达方式、验证读数和风险。
   - 区分“直接建模”“近似建模”“暂不能建模”。

5. **生成 markmap HTML**
   - 用 markmap 展示文献、实体、证据链和建模计划。
   - 避免只输出长表格；表格内容应转成可展开节点。

6. **验证**
   - 浏览器打开 HTML，确认 markmap 渲染。
   - 至少核对每篇论文 3 个核心事实。
   - 对每个跨论文连接写明证据类型，避免硬拼。

## markmap 模板

```markdown
# <主题>：虚拟细胞多论文证据网络

## 1. 文献角色
### 1.1 <Paper A>
- **角色**
  - 工程应用 / 综述背景 / 机制验证 / 数据来源。
- **对模型的贡献**
  - 可用信息。
### 1.2 <Paper B>
- **角色**
  - ...

## 2. 共同实体
### 2.1 基因/蛋白
- **<gene/protein>**
  - 出现论文、功能、建模意义。
### 2.2 代谢物
- **<metabolite>**
  - 出现论文、通路位置、模型意义。
### 2.3 通路
- **<pathway>**
  - 连接哪些论文。
### 2.4 表型
- **<phenotype>**
  - 数据来源和比较条件。

## 3. 证据连接
### 3.1 支持关系
- **<Paper A> -> <Paper B>**
  - 支持什么机制。
### 3.2 扩展关系
- **<Paper A> -> <Paper C>**
  - 从工程现象扩展到调控/互作/稳态。
### 3.3 冲突或不可直接合并
- **<claim>**
  - 冲突原因：物种、条件、方法、尺度。

## 4. 综合建模假设
### 4.1 假设 1
- **<hypothesis>**
  - 支持文献、涉及实体、模型表示、验证读数。
### 4.2 假设 2
- **<hypothesis>**
  - ...

## 5. 模型实现计划
### 5.1 第一轮：可直接模拟
- **knockout / exchange / objective**
  - 具体动作。
### 5.2 第二轮：近似模拟
- **overexpression / regulation / transporter**
  - 参数扫描方案。
### 5.3 第三轮：需扩展模型
- **protein interaction / enzyme complex / dynamic regulation**
  - 建议方法。

## 6. 风险与缺口
### 6.1 证据风险
- **跨物种套用**
  - 说明。
### 6.2 参数缺口
- **<parameter>**
  - 下一步查证。
```

## 质量标准

- 每条跨论文连接都要说明为什么能连。
- 不把综述中的多物种机制直接当作 E. coli 事实。
- 不把蛋白互作直接等同于稳定复合体，除非原文明确证明。
- 输出必须能作为项目建模假设会的讨论材料。
